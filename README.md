## Задание

Цель работы: овладеть полным циклом разработки модели сверточной нейронной сети, начиная с подготовки данных и заканчивая проверкой и анализом результатов на реальных данных.

Этапы выполнения работы:

1. Анализ задачи и постановка задачи:

• Изучите задачу и сформулируйте математическую постановку задачи относительно возможных категорий аватаров (согласно варианту).
• Проанализируйте, какие параметры и признаки могут быть важны для классификации аватаров (например, присутствие лиц, тип фона, стиль изображения).

2. Подготовка исходных данных:

• Используйте несколько готовых крупных датасетов (например, ImageNet, CelebA, COCO, Open Images) для получения категорий изображений, подходящих для обучения (согласно варианту).
• Создайте новый датасет, выбрав изображения, соответствующие различным категориям аватаров. Вам нужно будет отобрать изображения по категориям (согласно варианту).
• Осуществите предобработку собранных данных: приведение изображений к единому размеру (например, 224x224 пикселя), нормализация изображений, аугментация данных для увеличения объема и разнообразия.

3. Проектирование и обучение модели:

• Разработайте архитектуру модели (CNN), которая будет способна классифицировать аватары по категориям.
• Начните с базовых архитектур, таких как ResNet, VGG или MobileNet. Используйте transfer learning, если это необходимо для ускорения процесса обучения.
• Обучите модель на вашем собранном датасете.
• Подберите гиперпараметры сети.
• Используйте различные методы оптимизации (например, Adam, SGD) и следите за метриками качества (точность, F1-score, полнота, точность).
• Оцените модель на тестовой выборке, убедитесь, что она правильно классифицирует различные категории аватаров.

4. Проверка на новых данных (аватарах ВКонтакте)

• Возьмите датасет с аватарами пользователей ВКонтакте, которые не были использованы в обучении.
• Примените обученную модель для классификации новых аватаров.
• Сравните результаты, выявив возможные неточности или аномалии. Оцените, насколько успешно ваша модель справляется с классификацией на новых данных.

5. Анализ результатов и ответы на вопросы психолога

• Провести анализ результатов классификации и ответить на социальнопсихологические вопросы, предложенные психологами.

Вопросы для анализа:

1. Каково частотное распределение различных видов аватаров в молодежной группе пользователей ВК?
2. Существуют ли гендерные различия в частотном распределении?
3. Существуют ли различия, связанные с типом образования пользователя (техническое, естественнонаучное, гуманитарное)?
4. Каковы статистически достоверные предпочтения в выборе аватара в общей выборке, какие предпочтения у пользователей возраста 18-35 лет?
5. Каковы предпочтения в мужской группе выборки?
6. Каковы предпочтения в женской группе?
7. Связаны ли достоверные предпочтения с типом образования (техническое, гуманитарное, естественнонаучное образование)?

Для ответов на вопросы используйте статистические методы анализа данных. Постройте гистограмму распределения типов аватаров.

6. Подготовка отчет согласно шаблону.

Дополнительные рекомендации:

• Используйте предобученные модели с возможностью дообучения (transfer learning), если объем данных ограничен.
• Обратите внимание на балансировку данных по категориям (особенно по редким категориям аватаров).

## Quick start
1. Create virtualenv and install:
   python -m venv venv
   venv/Scripts/activate (deactivate)
   pip install -r requirements.txt

2. Provide source image folders for each class. Examples:
   - MIT Places: скачать вручную с https://places.csail.mit.edu/ (большой)
   - Cityscapes: https://www.cityscapes-dataset.com/ (регистрация/скачивание)
   - Kaggle: можно использовать скрипт + kaggle API (скопируй kaggle.json в ~/.kaggle/)

3. Run:
   python prepare_dataset.py --sources_landscape raw_data/landscape --sources_urban raw_data/urban --sources_fantasy raw_data/fantasy --out_dir dataset --n_per_class 5000 --val_ratio 0.2 --face_blur

## Notes
- If source folders don't contain enough images, скрипт будет многократно аугментировать имеющиеся.
- Face detection uses OpenCV Haar cascades (встроенный файл в OpenCV).
- Output images are saved as JPEG 224×224.
